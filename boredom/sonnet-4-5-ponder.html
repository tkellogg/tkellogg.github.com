Classical view (everyday intuition)

A tiny, localized "thing" - like a microscopic billiard ball with definite position and momentum moving through space.

2. Common Causes & Solutions

Null Pointer Dereference

// Bad
char *ptr = NULL;
*ptr = 'a';  // SEGV!

// Good
char *ptr = malloc(10);
if (ptr != NULL) {
    *ptr = 'a';
    free(ptr);
}

Buffer Overflow

// Bad
char buffer[10];
strcpy(buffer, "This is way too long!");  // SEGV!

// Good
char buffer[100];
strncpy(buffer, "Safe string", sizeof(buffer) - 1);
buffer[sizeof(buffer) - 1] = '\0';

Accessing Freed Memory

// Bad
char *ptr = malloc(10);
free(ptr);
*ptr = 'a';  // SEGV!

// Good
char *ptr = malloc(10);
*ptr = 'a';
free(ptr);
ptr = NULL;  // Prevent use-after-free

Stack Overflow

// Bad
void recursive() {
    recursive();  // Infinite recursion -> SEGV
}

// Good
void recursive(int depth) {
    if (depth > 1000) return;
    recursive(depth + 1);
}

Uninitialized Pointer

// Bad
int *ptr;
*ptr = 42;  // SEGV!

// Good
int *ptr = malloc(sizeof(int));
if (ptr) {
    *ptr = 42;
    free(ptr);
}

Why C Doesn't Have Namespaces

Historical Context

C was created in the early 1970s when programming language design was much simpler. Namespaces as a language feature didn't really exist yet - they emerged later as programs grew larger and more complex.

The Real Reasons

1. Philosophy of Simplicity

C was designed to be a minimal, low-level language close to assembly. Adding namespaces would have added complexity to:

  • The compiler
  • The linker
  • The language specification
  • The mental model

2. Single Global Namespace Was "Enough"

Early C programs were relatively small. Name collisions weren't seen as a huge problem when you had:

  • A few dozen functions
  • Simple libraries
  • Small teams

3. Workarounds Existed

C developers adapted with conventions:

// Prefixing (still used today)
sqlite3_open()
gtk_window_new()
json_parse()

// Static keyword for file-local scope
static void internal_helper() { }

// Struct-based pseudo-namespaces
typedef struct {
    void (*init)(void);
    void (*cleanup)(void);
} MyModule;

4. Backwards Compatibility

By the time namespaces became popular (C++ in the 80s), C had massive existing codebases. Adding namespaces would break everything.

Modern Pain Points

Yeah, it sucks when you have to:

#include <windows.h>
#include <mylib.h>
// ERROR: both define "Rectangle"

But C's simplicity is also its strength - it's stable, predictable, and will compile the same way in 2050.
