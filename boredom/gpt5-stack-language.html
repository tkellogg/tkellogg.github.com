Moving on: tiny stack-based language sketch

Name: Grain (small pieces that add up)

Idea

  • Concise, concatenative, Forth-like.
  • Everything is a word; words consume/produce values on a data stack.
  • Quotations (blocks) are first-class.

Syntax at a glance

  • Tokens separated by whitespace.
  • Literals: 42, -3.14, "hello", true, false.
  • Quotation: [ ... ] pushes a block.
  • Definition: : name … ; defines a new word.
  • Comments: \ to end of line. ( … ) inline comment.
  • Stack effect notation in docs only: word ( in… -- out… )

Core types

  • int, float, bool, string, quote (a block).

Core words (minimal set)

  • Stack: dup (a -- a a), drop (a --), swap (a b -- b a), over (a b -- a b a), rot (a b c -- b c a), nip (a b -- b)
  • Arithmetic: + - * / mod (all numeric; / is float if needed)
  • Comparisons: = != < > <= >= (a b -- bool)
  • Logic: and or not
  • I/O: . (a --) prints top, .s shows stack non-destructively, cr newline
  • Control (quotes):
    • call (q -- …) executes a quotation
    • if (qtrue qfalse cond -- …) executes one of two quotations
    • when (q cond -- …) executes q if cond
    • while (qbody qcond -- …) repeats qbody while qcond leaves true
  • Definitions/vars:
    • : name … ; define word
    • var name ( -- addr) creates a cell; @ (addr -- val) fetch; ! (val addr --) store
    • value name ( val -- ) defines a constant word with current top
  • Utilities: length (str -- n), emit (charcode --), str+ (s1 s2 -- s)

Semantics

  • Interpreter reads tokens left to right.
  • Literal pushes itself. Known word executes immediately.
  • Quotations capture sequence until matching ] and push it as a value.
  • Errors on stack underflow or type mismatch.
  • Words are late-bound; definitions override earlier ones.

Grain interpreter: minimal architecture (pseudo-code)

  • State
    • stack: list
    • dict: map<string, Word> // builtins + user defs
    • memory: map<int, Value> // for var cells; addr is int
    • nextAddr: int = 0
  • Value (tagged union)
    • Int(n), Float(x), Bool(b), Str(s), Quote(list), Addr(int)
  • Token
    • Literal(Value) | WordName(string) | QuoteStart | QuoteEnd
  • Tokenize(src)
    • Split by whitespace.
    • Strings: "…", accept escapes.
    • Comments: "" to end-of-line; "( … )" skip until matching ).
    • [ pushes QuoteStart; ] pushes QuoteEnd.
    • Numbers -> Int/Float; true/false -> Bool; else WordName.
  • Parse stream to executable token list
    • On QuoteStart: capture nested tokens until matching QuoteEnd, recursive.
    • Wrap captured list into Value.Quote(...) and emit as Literal.
  • Execute(programTokens)
    • For each token:
      • Literal(v) -> push(v)
      • WordName(w):
        • if in dict: dict[w].fn(state)
        • else error UnknownWord(w)
  • Error strategy
    • On error: throw with
      • type: StackUnderflow | TypeError | UnknownWord | MismatchedBracket | DivisionByZero | RecursionLimit
      • context: word name, expected stack effect, got types, current stack snapshot (top…bottom, truncated).

    • rot: (a b c -- b c a)

    • .: (a --) print top

    • cr: ( -- ) print newline
  • Arithmetic

  • Compare/logic

  • Quotes/control

Implementation notes

  • Word is late-bound: definitions added to dict override earlier.
  • Quotes are stored as token vectors; executing a quote runs the same interpreter over that token list with current dict (late binding).
  • Tail recursion not required; recursion depth limit prevents runaway.

Higher-order words (range/count flavored)

  • times: (n q --)
    • Runs q exactly n times; q takes no inputs, provides any outputs which remain on stack cumulatively.
    • Pseudo:
      • pop q, n
      • for i in 0..n-1: exec(q)
  • timesi: (n q --)
    • Like times, but pushes the loop index i before each call so q can consume it.
    • Contract: q should expect (i -- …).
    • Pseudo:
      • for i in 0..n-1: push Int(i); exec(q)
  • reduce-range: (init n q -- result)
    • Folds i = 0..n-1; q takes (acc i -- acc)
    • Pseudo:
      • pop q, n, acc
      • for i in 0..n-1:
        • push acc, i; exec(q); acc = pop()
      • push acc
  • map-range: (n q -- …n results…)
    • Calls q with (i -- val) for i=0..n-1, pushing each result; leaves n values on stack in increasing i order.
  • filter-range: (n q -- k …k kept indices…)
    • Calls q with (i -- Bool); pushes count k then the kept indices (or use a temp list then emit); simple but handy.

Errors and control (Forth-style CATCH/THROW)

  • throw (err --) raises an error value (typically a string). Aborts to nearest catch.
  • catch (q -- 0 | err) runs q in protected mode. On success pushes 0; on error pushes the error value.
  • try (qbody qcatch -- …)
    • Runs qbody under catch. If it errors, leaves the error on stack and runs qcatch.
    • Stack contract: qcatch should expect (err -- …).
  • assert (cond msg --)
    • If cond is false, throw msg; else drop msg and continue.

Suggested error values

  • Simple: strings like "Division by zero".
  • Rich: strings prefixed with a word, e.g., "TypeError:/ expects numbers".
  • Interpreter-originated errors still provide structured diagnostics to console; the value thrown into the data stack is a concise message string.

Result-tag pattern (tagged outcomes without exceptions)

  • ok (v -- "ok" v)
  • err (e -- "err" e)
  • is-ok? (tag val -- bool)
  • on (qok qerr tag val -- …)
    • If tag="ok", runs qok with v on stack.
    • If tag="err", runs qerr with e on stack.
  • unwrap-or (tag val fallback -- val’) returns v if ok, else fallback.
